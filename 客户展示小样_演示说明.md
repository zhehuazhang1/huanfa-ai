用途：给客户做第一轮产品方向展示，确认需求、争取认可、推进报价和签约。

## 2. 这版小样展示什么

这版不是最终小程序，不连接真实后台，目的是让客户看到产品长什么样、怎么用、解决什么问题。

主要展示：

1. 顾客端AI发型推荐。
2. 男客、女客、通用发型发色区分。
3. 顾客预约流程。
4. 会员中心雏形。
5. 15家门店老板看板。
6. 后续落地流程。

---

## 3. 推荐演示顺序

### 第一步：先讲客户痛点

可以这样说：

```text
现在顾客进店前最大的问题是，不知道自己适合什么发型，前台和发型师也要花很多时间沟通。
这个小程序第一步就是帮顾客先做初步筛选，把适合的发型、发色、发型师推荐出来。
```

### 第二步：演示AI推荐

在手机模拟界面里：

1. 点击“AI推荐”。
2. 选择“女客”。
3. 选择“中发”。
4. 选择“韩系”。
5. 选择“接受染发”。
6. 点击“生成AI推荐”。

讲解重点：

```text
这里不是让AI随便乱推荐，而是先根据性别、发长、风格、是否接受染发做规则过滤。
比如女客不会推荐男士发型，男客不会推荐明显女款发型。
AI主要负责把推荐理由讲清楚，让顾客更容易接受。
```

### 第三步：演示男女区分

把“女客”切换成“男客”，再点击“生成AI推荐”。

讲解重点：

```text
系统会自动切换成男士发型逻辑。
这一点很重要，因为美发场景里男客和女客的发型、发色、服务时长、适合发型师都不一样。
```

### 第四步：演示预约

点击“预约”。

展示：

1. 门店选择。
2. 发型师选择。
3. 预约时间。
4. 确认预约。

讲解重点：

```text
顾客不用反复加微信沟通，能直接提交预约。
后台会绑定门店、发型师、套餐和时间，避免同一个发型师同一时间重复预约。
```

### 第五步：演示会员

点击“会员”。

讲解重点：

```text
服务完成后，顾客会沉淀到会员体系里。
后续可以做积分、生日优惠、流失召回，提高复购。
```

### 第六步：讲老板看板

看右侧“15家门店老板看板”。

讲解重点：

```text
老板可以看所有门店，店长只能看自己门店。
未来可以看到预约量、到店量、营收、客单价、门店排行、发型师排行。
```

## 4. 和客户沟通时要强调的价值

### 4.1 对顾客

1. 不知道选什么发型时，有AI给参考。
2. 推荐更贴近性别、发长、风格和烫染接受度。
3. 可以直接预约，减少沟通成本。

### 4.2 对门店

1. 前台咨询压力降低。
2. 发型师匹配更清楚。
3. 预约流程更标准。
4. 顾客数据能沉淀为会员。

### 4.3 对老板

1. 15家门店统一管理。
2. 能看每家店经营数据。
3. 能看门店排行和发型师表现。
4. 后续能做会员复购和营销。

---

## 5. 不要过度承诺的话

演示时不要说：

```text
AI一定能百分百推荐最适合发型。
```

建议说：

```text
AI先帮顾客做初步筛选和沟通，提高到店前的决策效率，最终方案仍以发型师面诊为准。
```

不要说：

```text
虚拟换发第一版一定上线。
```

建议说：

```text
虚拟换发可以做，但涉及顾客照片和人脸隐私，建议放到第二或第三阶段，先把预约和推荐闭环跑通。
```

不要说：

```text
美团、抖音、大众点评马上都能接。
```

建议说：

```text
平台接口可以预留，真实接入要看平台开放权限和客户账号资质。
```

不要说：

```text
储值卡可以马上做。
```

建议说：

```text
微信支付可以后续接入，但储值卡涉及预付款合规，建议先做积分和优惠券。
```

## 6. 客户可能会问的问题

### Q1：这个是不是已经能上线？

回答：

```text
这版是展示小样，用来确认产品方向和核心流程。
真正上线版还需要开发后端、数据库、微信小程序、权限、AI推荐和部署。
```

### Q2：AI推荐准不准？

回答：

```text
第一版会采用规则过滤加AI解释。
规则保证不跑偏，比如男客不推荐女款，女客不推荐男款。
AI负责把推荐理由讲清楚，最终发型仍由发型师确认。
```

### Q3：15家门店能不能撑住？

回答：

```text
可以，但一开始就要按多门店系统设计。
每条业务数据都带门店ID，老板看全部，店长看本店，发型师看本人。
```

### Q4：照片会不会泄露？

回答：

```text
第一阶段不保存顾客照片。
如果后续做虚拟换发，也会先弹隐私授权，并且不把顾客照片保存到数据库。
```

### Q5：多久能做出第一版？

回答：

```text
建议先确认展示小样和功能范围，再评估周期。
如果第一版只做核心预约、AI推荐、门店权限和基础看板，周期会更可控。
```

## 7. 建议给客户看的三份材料

建议按这个顺序给客户：

1. `客户展示小样.html`：先让客户看到产品感觉。
2. `美发AI小程序_项目可行性分析报告.md`：说明项目可行、风险可控。
3. `美发AI小程序_AI执行落地计划_15店版.md`：说明后续怎么开发和验收。

---

## 8. 本轮展示目标

这次不要急着把所有功能都讲完。

本轮目标是让客户确认三件事：

1. 是否认可AI推荐 + 预约 + 多门店管理这个方向。
2. 是否认可第一阶段先做核心闭环，不把虚拟换发和外部平台接口塞进首版。
3. 是否愿意提供发型、发色、套餐、发型师、门店资料，用于后续正式开发。
