## 一、成本真相（已查实，地基中的地基）

- 生成走 **自建 Dify → 通义万相 `wan2.7-image`**（主力）+ 阿里云 ImageSeg 分割 + FaceBody 人脸 + OSS。
- **真实成本 ≈ ¥0.34 / 次试发**（1 次 = 3 张图）。
  - 依据：2026-06 账单 ¥17.10 ÷ 数据库实测 51 成功组 = ¥0.335/组；万相 ¥0.10/张。
- ⚠️ wan2.7 的 ¥0.10/张 大概率是**百炼促销价**，定价需留余量（按 ¥0.6 成本压力测试）。
- 6 月数据：发起 63 组 / 成功 51 / 失败 12（**失败率 19% 偏高，待查**）。

## 五、经营模块设计（专业版+，老板端开关）

### ① 主理人提成 · 基础版（定稿）
**决策**：基数=实收金额；一单两人（主理+助理，各自算）；主理人可自看业绩；~~AI 转化奖~~ 取消（但 `is_ai_converted` 保留作统计）。

**表结构**
```
service_items（扩展）: commission_type(percent/fixed/none) + commission_value
service_records（扩展）: assistant_id（助理，可空）
店铺设置: assistant_commission（助理统一一档，建议起步 5% 或 ¥10/单）
提成 = 实时按 service_records × 项目提成；月度按【人】汇总（主理单+助理单），可标已结算
```
**商家端**：① 项目管理（建项目填提成）② 结账录入（顾客+主理人+助理+项目+实收）③ 提成报表（老板看全店 / 主理人看自己）。
**联动**：结账录入 = 枢纽，一次喂订单+提成+CRM+（二期储值核销）；与现有绩效统计合并成"主理人中心"。

### ② 顾客储值 + 会员 / 次卡（专业版+，记账型）
**定位**：钱线下收，商家手动录入，系统只记账/核销，**不碰支付、无需 ICP**。产品需标注"经营数据由商家自行录入"。
```
customer_wallets（余额）+ wallet_transactions（充值/消费/赠送/退款流水）
membership_levels（等级/折扣）+ member_cards（次卡/疗程/等级卡，余次/有效期）+ card_redemptions（核销）
```
- 储值卡/次卡/门店套餐 = 同一套记账逻辑；线下付，系统核销。
- 退款 = 手填负流水。

## 十、系统体检发现（2026-06-11，ROADMAP 之外新增）

**🟡 P1 · 体验 / 一致性**
