# 前端小程序联调状态

当前已经新增三类前端/展示骨架：

1. `miniapp-customer`：顾客端微信小程序。
2. `miniapp-merchant`：商家端微信小程序。
3. `平台后台展示小样_v1.html`：平台方后台展示小样。

## 顾客端已完成

目录：`miniapp-customer`

已覆盖：

1. 首页。
2. 微信登录演示用户。
3. 到店扫码确认免费资格。
4. 底部入口：发型灵感、我的订单、AI咨询。
5. AI造型方向：女性、男性、中性。
6. 左侧分类：热门、长发、中发、短发、发色。
7. 右侧发型/发色素材展示。
8. 选择发型、发色，支持只选发型、只选发色、发型+发色。
9. 隐私授权。
10. 临时上传URL。
11. AI生成。
12. 结果页三张图滑动。
13. 推荐3位主理人，默认第一位。
14. 下单。
15. 我的订单。
16. AI咨询。

展示模式默认使用同步生成：

```js
useSyncGenerate: true
```

正式上线建议切到排队模式：

```js
useSyncGenerate: false
```

## 商家端已完成

目录：`miniapp-merchant`

已覆盖：

1. 工作台：订单、AI试发、AI转化、成交金额。
2. 订单页：按状态筛选订单。
3. 订单流转：确认预约、确认到店、开始服务、完成服务、取消。
4. 完成服务时读取商家服务项目，并输入真实成交金额。
5. AI次数页：给顾客赠送1次AI试发。
6. AI次数页：给主理人追加当天赠送额度。
7. 赠送转化：赠送数、使用数、下单数、成交金额。
8. 业绩页：完成服务、成交金额、AI转化成交。
9. 主理人排行。
10. 服务项目统计。
11. 图库管理页：按女性、男性、中性查看发型/发色，并可新增演示素材。
12. 主理人管理页：查看主理人、擅长方向、技能标签，并切换在店/忙碌/不在店。
13. 服务项目页：查看剪发、染发、烫发、造型、护理项目，可筛选全部/启用/停用，并可新增、停用、重新启用。
14. 业绩页：增加按业务分类展示，支持剪发、染发、烫发、造型、护理汇总。

1. 顾客端登录使用演示 openid，真实上线要接微信 `code2Session`。
2. 商家端身份使用固定 `staffId=2`，真实上线要接商家登录和权限。
3. 顾客端图片使用远程演示图，正式上线要替换为商家上传素材和 OSS 临时图。
4. 微信支付仍是 Mock，正式商业MVP必须接微信支付。
5. AI真实生成还没有接入真实通义万相 API Key。
