# POC 验证清单

## 测试素材

至少准备：

```text
男性自拍 3 张
女性自拍 3 张
中性风格自拍 3 张
```

每类都测试：

```text
只换发型
只换发色
发型 + 发色
```

## 记录字段

每次测试记录：

```text
测试编号
方向：male/female/neutral
输入照片编号
选择发型
选择发色
是否只换发型
是否只换发色
是否发型发色都换
是否生成3张
是否像本人
是否改脸
是否只改头发
发色是否准确
发型是否可接受
生成耗时秒
真实API成本
是否可给客户展示
备注
```

## 通过标准

建议进入 MVP 的最低标准：

```text
生成成功率 >= 85%
平均生成耗时 <= 45 秒
明显改脸率 <= 10%
可展示效果占比 >= 70%
单次3图真实成本在可转售范围内
```

## 不通过处理

如果不通过：

1. 优先调 Dify 提示词。
2. 再调输入发型/发色标签。
3. 再评估是否更换通义万相参数或模型版本。
4. 不要先做完整小程序开发，避免把钱花在错误能力上。
