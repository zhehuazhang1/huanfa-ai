# 微信小程序本地联调手册

这一步的目标是把项目从“代码已完成”推进到“能在微信开发者工具里打开、点击、演示”。

## 1. 启动后端

在项目目录运行：

```powershell
powershell -ExecutionPolicy Bypass -File .\启动本地后端.ps1
```

成功后会看到：

```text
Backend started: http://127.0.0.1:8000/health
```

浏览器或接口工具访问：

```text
http://127.0.0.1:8000/health
```

应返回：

```json
{
  "status": "ok",
  "database": "ok"
}
```

后端启动后，可以先跑真实 HTTP 联调：

```powershell
python backend/scripts/live_miniapp_smoke.py http://127.0.0.1:8000
```

成功时会看到：

```text
PASS live miniapp smoke
```

停止后端：

```powershell
powershell -ExecutionPolicy Bypass -File .\停止本地后端.ps1
```

## 2. 导入顾客端小程序

打开微信开发者工具：

1. 选择“导入项目”。
2. 项目目录选择：

```text
C:\Users\73177\Documents\美发\miniapp-customer
```

3. AppID 可先选择测试号或小程序测试 AppID。
4. 勾选“不校验合法域名、web-view 域名、TLS版本以及HTTPS证书”。
5. 编译运行。

顾客端要点一遍：

1. 首页。
2. 登录演示用户。
3. 到店扫码确认。
4. 开始AI造型。
5. 选择女性、男性或中性。
6. 选择发型、发色，或只选其中一个。
7. 同意隐私授权。
8. 生成三张图。
9. 横向滑动查看结果。
10. 选择推荐主理人。
11. 下单。
12. 进入“我的订单”查看订单。

## 3. 导入商家端小程序

打开微信开发者工具再导入一个项目：

```text
C:\Users\73177\Documents\美发\miniapp-merchant
```

商家端要点一遍：

1. 工作台。
2. 订单处理。
3. 订单状态流转：确认、到店、服务、完成。
4. 完成服务时选择服务项目，并输入成交金额。
5. AI次数页：赠送顾客AI次数。
6. AI次数页：给主理人追加赠送额度。
7. 业绩页：查看成交、AI转化、主理人排行。
8. 图库管理：查看女性、男性、中性发型发色。
9. 主理人管理：切换在店、忙碌、不在店。
10. 服务项目：新增或停用服务项目。

## 4. 打开平台后台小样

直接双击：

```text
C:\Users\73177\Documents\美发\平台后台展示小样_v1.html
```

点击“读取本地后端”，如果后端已启动，会读取真实本地数据：

- 客户数。
- 门店数。
- AI剩余额度。
- 平台预估毛利。
- 每个客户的门店、额度、每日上限。

## 5. 每次改完先验收

```powershell
powershell -ExecutionPolicy Bypass -File .\一键验收.ps1
```

看到 `All checks passed.` 再继续联调。

## 6. 当前仍是 Mock 的部分

本地联调阶段先不接真实外部服务：

- AI生成：Mock Dify。
- 图片上传：Mock 临时 URL。
- 微信支付：Mock 支付。
- 飞书同步：Mock 同步。
- 微信手机号真实登录：演示 openid。

这一步跑通后，下一步才接真实 Dify + 通义万相 + OSS，开始真实换发型发色 POC。
