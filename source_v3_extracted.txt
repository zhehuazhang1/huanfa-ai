5.3 权限中间件核心逻辑
# 每个API请求都经过权限过滤
@app.middleware('http')
async def rbac_middleware(request, call_next):
    user = decode_jwt(request.headers.get('Authorization'))
    if user.role == 'stylist':
        request.state.filter_stylist_id = user.stylist_id
    elif user.role == 'manager':
        request.state.filter_store_id = user.store_id
    elif user.role == 'boss':
        pass  # 无过滤
    return await call_next(request)
⚠️ 注意：所有涉及store_id过滤的API，必须在服务端强制执行，不能仅依赖前端传参。前端传的store_id只作参考，服务端以JWT中的store_id为准。

8.3 低库存预警逻辑
# 每天08:00执行
def run_stock_alert():
    low_stock = db.query(
        'SELECT * FROM products WHERE stock <= low_stock_alert AND is_active=1'
    )
    for product in low_stock:
        manager = get_store_manager(product.store_id)
        send_subscribe_msg(manager.openid, '库存预警', {
            '产品': product.name,
            '当前库存': product.stock,
            '预警阈值': product.low_stock_alert
        })

第12章：⚠️ 重要注意事项汇总（AI必读）
本章汇总了项目中所有重要的风险点、易错点和强制要求。AI执行每个模块时，必须先检查本章相关条目。
12.1 服务器与基础设施
⚠️ 注意：服务器内存必须是16GB。8GB内存在Dify+MySQL+Redis+Nginx+Weaviate全部运行时会OOM崩溃。选购服务器时这是第一优先级。
⚠️ 注意：DIFY_WORKER_NUM=4 必须在.env中配置。默认值1在高并发时会让所有工作流请求排队，严重影响顾客体验。
⚠️ 注意：HTTPS是微信小程序的强制要求。开发环境可用，生产环境必须有有效SSL证书和域名，不接受IP直接访问。
12.2 AI工作流与知识库
⚠️ 注意：Dify知识库不自动同步数据库。每次发型/套餐/发型师数据变更后，必须手动或通过脚本触发知识库更新，否则AI推荐结果会基于过时数据产生错误推荐。
⚠️ 注意：Dify API Key和Face++ AppKey绝对不能出现在小程序前端代码中。前端代码可被反编译，泄露API Key会造成费用损失和安全风险。所有密钥仅保存在服务器环境变量中。
💡 建议在服务器.env文件中统一管理：DIFY_API_KEY、FACEPP_API_KEY、OSS_ACCESS_KEY等所有密钥。
12.3 隐私合规
⚠️ 注意：顾客照片绝对不能存储。Face++虚拟换发功能必须使用前端SDK直传方式，照片不经过后端服务器，数据库中不保存任何照片URL。只保存style_id和color_id两个文本字段。
⚠️ 注意：隐私弹窗不能跳过。在调用任何拍照/选图功能前，必须先弹出隐私说明弹窗，用户主动点击「同意」才能继续。这是法律合规要求。
12.4 微信小程序限制
⚠️ 注意：微信小程序不支持iframe。所有AI对话界面必须自行开发聊天UI，通过调用Dify Chat API实现消息交互，不能嵌入任何网页。
⚠️ 注意：微信订阅消息不能强制批量发送。只能在用户主动操作时请求订阅授权，用户明确同意后才能发送。需提前申请「本地服务-美容美发」长期订阅资格，审核需1-4周，必须在上线前2个月申请。
12.5 财务与法律风险
⚠️ 注意：充值储值（余额充值/储值卡）有预付款合规风险，需申请相关资质。MVP阶段强烈建议改用积分+优惠券替代，规避法律风险。如客户坚持做储值功能，必须先咨询法律顾问。
12.6 运营注意事项
⚠️ 注意：飞书和MySQL的数据同步必须测试。飞书是展示和录入工具，MySQL是系统主数据库，两边必须保持一致。Webhook同步脚本需要单独部署和测试，确认每次飞书录入后MySQL能正确更新。
⚠️ 注意：外部平台预留接口当前返回{status: not_connected}。这是正确行为，说明接口已预留但未接入。接入前不要修改这些接口的路由路径，以保证未来对接时URL不变。
💡 新增管理员：在users表中插入用户记录，设置role和store_id字段即可。不需要改代码。
💡 新增门店：在stores表中插入一条记录，相关API会自动支持新门店的数据隔离。
