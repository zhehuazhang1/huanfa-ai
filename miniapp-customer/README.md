# 顾客端微信小程序联调骨架

这个目录是顾客端小程序第一版联调骨架，用来跑通客户确认过的主流程：

1. 微信登录演示用户。
2. 到店扫码确认免费资格。
3. 选择女性/男性/中性方向。
4. 选择发型、发色，支持只选发型、只选发色、发型+发色。
5. 拍照/自拍前记录隐私授权。
6. 创建临时上传URL并把顾客选择的自拍上传到临时存储。
7. 调用AI生成，处理完成后后端主动清理临时自拍。
8. 结果页三图滑动、推荐3位主理人。
9. 下单。
10. 我的订单。
11. AI咨询。

## 打开方式

用微信开发者工具导入 `miniapp-customer` 目录。

本地后端默认地址在 `app.js`：

```js
apiBaseUrl: "http://127.0.0.1:8000"
```

如果后端部署到服务器，把这里改成 HTTPS 域名。

## 展示模式

当前 `app.js` 里：

```js
useSyncGenerate: true
```

展示阶段直接调用 `/ai/style/generate`，方便客户看完整闭环。

正式上线建议改为：

```js
useSyncGenerate: false
```

然后由后端 Worker 消费 `/ai/style/enqueue` 创建的任务。

## 注意

1. 当前图片使用远程演示图，微信开发者工具里需要勾选“不校验合法域名”。
2. 正式上线时，商家照片、发型缩略图、AI生成临时图都应走已备案 HTTPS 域名或 OSS 临时链接。
3. 在微信公众平台配置后端 HTTPS 域名为 request 合法域名，并把 OSS Bucket HTTPS 域名配置为 request 合法域名，供自拍 PUT 上传使用。
4. OSS 的 `temp/` 路径应配置生命周期自动删除规则，作为主动清理失败时的兜底。
5. 目前登录使用演示 openid，真实微信登录需要后端接入 `code2Session`。
6. 后端测试以根目录 `一键验收.ps1` 的实时输出为准。
