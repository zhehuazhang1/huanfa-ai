# 项目验收与联调说明

本文档记录当前项目“能不能跑通”的自动检查方式，适合每次改完后端、小程序页面、商家端页面后执行。

## 0. 一键验收

最简单方式：

```powershell
powershell -ExecutionPolicy Bypass -File .\一键验收.ps1
```

它会依次检查：

- 小程序 JSON 配置。
- 后端单元测试。
- 顾客、商家、平台、SaaS隔离端到端验收。
- Dify 工作流输出契约。
- 如果本地后端已启动，还会额外跑真实 HTTP 小程序联调。

## 1. 后端单元测试

命令：

```powershell
$env:PYTHONPATH='backend'
python -m pytest backend/tests -q
```

检查范围：

- AI生成成功才扣次数，失败和超时不扣次数。
- 免费、赠送、付费三种AI资格。
- 单用户不能同时创建多个生成任务。
- 门店、租户、平台并发和每日限制。
- 排队时长 `queue_wait_seconds` 和生成耗时 `generate_duration_seconds`。
- 顾客看不到平台真实API成本。
- 租户隔离：租户2查不到租户1订单、支付单、AI任务。
- 发型发色库必须覆盖男性、女性、中性。
- Dify 返回少于3张图、slot错误、缺少临时图片URL时必须被识别为失败。

## 2. 端到端验收脚本

命令：

```powershell
python backend/scripts/acceptance_check.py
```

脚本会自动跑完整链路：

1. 后端健康检查。
2. 男性、女性、中性发型库检查。
3. 顾客登录。
4. 顾客隐私授权。
5. 到店扫码获得免费AI次数。
6. 创建自拍临时上传地址。
7. 选择发型和发色。
8. AI生成三张图。
9. 结果页返回三张滑动图片和三位推荐主理人。
10. 顾客选择主理人并下单。
11. 商家查看订单。
12. 商家完成服务并记录成交金额。
13. 商家业绩统计更新。
14. 平台查看AI使用量、真实成本、客户结算金额。
15. 平台客户看板返回每个客户的门店、额度、AI成功次数和真实成本。
16. 创建第二个租户，并验证第二个租户绝对查不到第一个租户的订单和AI任务。

成功时会输出：

```text
PASS acceptance check: customer, merchant, platform, and SaaS isolation are OK
```

## 2.1 真实 HTTP 小程序联调

先启动后端：

```powershell
powershell -ExecutionPolicy Bypass -File .\启动本地后端.ps1
```

再运行：

```powershell
python backend/scripts/live_miniapp_smoke.py http://127.0.0.1:8000
```

这个脚本会像微信小程序一样真实请求本地后端，检查登录、扫码、生成、下单、商家完成服务和平台看板。

## 3. Dify 工作流输出校验

命令：

```powershell
python backend/scripts/validate_dify_result.py backend/fixtures/dify_success_result.json
```

检查范围：

- Dify 输出必须是 JSON。
- 成功结果必须返回 3 张图。
- 3 张图的顺序必须是 `main,natural,advanced`。
- 每张图必须有 `temp_image_url`。
- 必须返回平台可统计的 `internal_api_cost`。

如果真实 Dify 输出不合格，脚本会返回失败。业务生成中这类结果也会按失败处理，不扣客户 AI 次数。

## 4. 当前验收结论

当前本地 Mock 模式已经跑通：

- 顾客端核心流程可联调。
- 商家端核心流程可联调。
- 平台后台核心数据可联调。
- SaaS 租户隔离通过自动验收。
- 中性发型发色库已补齐基础演示数据。
- 商家端已补齐图库、主理人、服务项目管理入口。
- 平台后台已补齐客户汇总接口 `/platform/tenant-dashboard`。

## 5. 还不能自动验收的部分

以下部分需要真实账号、密钥或微信环境：

- 微信手机号真实授权登录。
- 微信支付真实下单、回调验签、退款。
- 自部署 Dify 工作流真实调用。
- 通义万相真实图生图效果和真实单次成本。
- 阿里云 OSS 真实上传。
- 飞书真实表格或多维表同步。

这些不影响当前代码主链路 Mock 联调，但正式上线前必须逐项换成真实配置并重新验收。
